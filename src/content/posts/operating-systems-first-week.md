操作系统课程最容易让人迷失在名词里。这一篇先只抓住一个问题：**应用程序如何请求操作系统替它完成工作？**

## 为什么需要系统调用

普通程序不能直接访问所有硬件资源。系统调用提供了一条受控边界，让程序可以请求文件、网络和进程管理等能力。

> 这不是完整的课程笔记，而是一份可以继续扩展的文章样板。

## 用户态与内核态

可以先把二者理解成不同权限等级。用户态运行应用程序；内核态运行操作系统核心代码。

```c
int fd = open("notes.txt", O_RDONLY);
read(fd, buffer, sizeof(buffer));
close(fd);
```

这些看起来像普通函数的调用，最终需要跨过权限边界进入内核。

### 一次调用发生了什么

1. 程序准备参数。
2. CPU 切换到内核态。
3. 内核检查请求并执行操作。
4. 结果返回用户态。
