## Day 1

### [Strands Robots](https://github.com/strands-labs/robots)
这个工作主要聚焦于构建一个统一的机器人智能开发框架。最终的目的，是让 Agent 通过自然语言就能较好地调用这些工具，从而非常顺利地完成从 Agent 到 Policy，再到 Backend，最后到 Robot 这四层的架构设计、训练和部署。它通过提供统一的接口，将这些内容统一到一个系统之中。这个工作没有什么的理论创新，主要是提供非常好用的工具，便于具身智能的科研和实际部署。

然后我们更加详细地拆解一下这四层架构。

这里的 Agent 层，并不是指具身机器人本身使用 Agent 策略来指导自己的行动。这里的 Agent 更多的是用于进行具身智能科研——科研人员通过与 Agent 交互，让 Agent 利用整个项目中设计出来的工具，去实现各种功能。

例如：
1. 收集各种数据
2. 调用各种 policy
3. 在不同的 backend 环境（如不同的 simulator、现实世界，以及各种具体的机器人产品终端 robot）中进行部署和实现

所以，这个 Agent 并不是机器人本身 policy 的一部分，而是一个辅助科研的工具 Agent。

我觉得这个工具是我接下来需要认真去学习的，认识它肯定会对科研有帮助。应该吧，毕竟我现在还没开始科研。万一实际上其实有更好的思路呢？我也不知道，但是我会尽量学习一下。

### Diffusion Tutorial

这个关于 Diffusion 的 Tutorial，首先简单讲解了 Diffusion 模型的大致概念，其中也包括 Flow Matching。这块我之前大概只懂一点皮毛，原理不是很懂，不过代码倒是会写。

这次 Tutorial 更多侧重于分析 Diffusion 模型的记忆（Memorization）与生成（Generation）之间的区别。

我们知道，生成模型在训练时的目的是为了尽量拟合出数据的真实分布。但如果模型的拟合能力无限强，最后训练出来的模型数据分布就会和训练数据（Training Data）完全一致。这样一来，我们随机生成一张图片，其实只是从训练数据中随便抽了一张出来。这显然不是我们想要的生成效果，而仅仅是“背诵”（Memorization）。

因此，Tutorial 从 Diffusion 及其背后的原理层面，分析了它是否会有 Memorization 的行为，以及 generation 和 Memorization 之间的关系，即在什么情况下它能做到泛化，而不是完全的死记硬背。

实际上，如果我们看 Flow Matching，在最简单的将速度的 L2 Loss 最小化的训练方式下，它其实是有闭式最优解（Closed-form Optimal Solution）的。如果我们要得到这个闭式解的公式，在最后的 Time Step 非常接近目标数据（Data）时，它会把我们精确地推到实际的训练数据点上，而不会推向其他地方。所以说，如果完全满足闭式解，模型就必然会走向 Memorization。

那为什么在实际应用中，我们的 Flow Matching 很多时候却能做到 Generation 呢？

Tutorial 中针对这个问题提出了两个方向（其实 Memorization 和过拟合问题差不多，而解决过拟合最常用的方法就是正则化 Regularization）：

1. 动态正则化（Dynamic Regularization）
比如训练到一定阶段，发现模型已经可以生成但又没有完全开始死记硬背时，就主动停止训练；或者通过调整权重、使用一些传统的正则化方法，来防止模型完全走向背诵。

1. 架构正则化（Architectural Regularization）
这主要源于我们模型的架构设计。例如我们常用的 U-Net 属于 CNN 网络，它天然具有平移同变性（Translation Equivariance），并且受到感受野（Receptive Field）的限制，这使得它在物理结构上就无法达到那个理论上的闭式最优解。实验也发现，在闭式最优解的约束中加入这些限制性因素后，结果和实际模型跑出来的效果差不多。所以，正是我们实际使用的 U-Net 模型架构限制了 Memorization，从而实现了 Generation。

不过，这目前还只是针对 U-Net 的结论。对于 ViT 这种 Inductive Bias较小的架构，他们团队目前还没有得出最终的结论。

总的来说，这个 Tutorial 偏向于理论分析，探讨了“背诵”与“生成”之间的关系，为 Diffusion 如何达成生成提供了一种理论层面的合理解释。不过从我个人实际技术的角度来看，感觉并没有特别大的实用价值。

### From Digital Agents to Physical Intelligence: The Agentic Harness as a Unifying Architectural Pattern

这个 SPRO（Digital Agents to Physical Intelligence）我觉得是本次我收获最大的，虽然它感觉没有讲到非常多深刻、前沿的东西，更多还是以普及知识为主，但对我的研究兴趣还是有一定的启发。

他首先讲解了一下 Digital Agent（或者说我们现在这种基于文本的、在互联网上或代码里的 Agent）所需要的一个非常强大的 Harness 系统。我们需要这个 Harness 系统来为 LLM 服务，这样才能更好地利用 LLM 的能力。因为大家发现，一个裸的 LLM 就算再强，如果没有配上一个很好的 Harness，实际的功能其实也不佳。

关于 Harness 需要哪些东西，他提到了以下几点：
1. **记忆与技能**：Agent 需要有 memory，也需要有 skill。
2. **多智能体协同（Multi-Agent Collaboration）**：这有很多种合作方式：
   (a) 串行单智能体。
   (b) 树状/星状控制：分别有一个管理者（Orchestrator），控制多个 sub-agent 运行，每个 sub-agent 都与这个主的 Orchestrator 进行通信。
   (c) 去中心化团伙：整个 Agent 群变成一个团，两两之间都可以通信。
   (d) 混合模式：在上述通信团的上面，再加一个主的 Orchestrator 来控制其他的 sub-agent。

在 LLM 之外，我们怎么用 LLM 来实现 Agent 其实非常重要。对于 Digital Agent 来说，你必须得有 Harness，才能够执行一些大型代码项目的管理，或者一些安全要求非常高的操作（比如金融领域的应用，它每一步的流程必须是固定的，需要 Harness 来引导 Agent 怎么做）。

这是他对 Digital Agent 里面 Harness 范式的一种解释。他认为 Harness 一定会存在，虽然以后的模样可能会更加泛化。最开始大家都只是做 Prompt Engineering（提示词工程）和 Context Engineering（上下文工程），现在又到了 Harness Engineering，外面甚至还有 Loop Engineering。虽然一步一步地，Harness 会变得越来越多，但它确实是必不可少的。

他的下半场分享主要是想把这个 Harness 推广到 Physical Intelligence（具身智能）里面。这其实是我最感兴趣的部分，结果我当时没看完，本来想等会议结束了再看回放，没想到会议一结束回放就看不了了。

不过，这确实给了我很多科研上的启发。他说 Harness 有用，我确实深有同感。特别是做更复杂的任务，我觉得具身智能（Physical Intelligence）可能会比 Coding 更需要 Harness。这也是我以后研究兴趣比较大的方向。

## Day2

### Invited talk: Towards AI Agents In the Real World
这个 tutorial 跟第一天的那个 expo workshop 的主题其实都是要做真实世界中的 agent。不过两者侧重点不同：第一天的那个更多侧重于 harness，而今天这个则更加侧重于 world model。

首先，我们需要 physical agent。比如说像《钢铁侠》中的 Jarvis 助手，你可以问它问题、向它布置任务。这个时候，它都需要对整个世界有所了解，才能直接操作现实世界，才能做到更多的事情。

但是，目前 LLM 无法直接支撑起一个 AI agent。因为它只对语言这种模态比较了解，对其他模态的了解还远远不够：对视觉了解不够，听觉了解可能还行但也不够准确，至于触觉、动力学等各种模态的了解就更不够了。如果把所有信息都转成语言再喂给模型，这是否是一个好的选择？目前看起来并不是很好。所以，我们更多的是要能够理解世界上更多种模态的 world model。

关于 world model，目前有不同的技术路线：

1. Generate World Model：
   Generative world model 就是要整个的，比如说把视频给预测出来，也就是要在各种传感器的模态上，把未来世界给预测出来。如果你都预测出来了，那我们就认为你对整个世界的各种规律都掌控得比较好。

2. JEPA (Joint Embedding Predictive Architecture) 模型：
   如果你对整个世界做视频生成或整体仿真，它跟人脑实际的感觉是很不一样的。人脑不会有那么多注意力去关注每一个东西的纹路，你不可能把所有细节都还原出来，也不可能预测出 100 片树叶会如何下落。但是你可能会注意到其中的一些核心细节，看得比较仔细，或者在语义上了解得更多一点。
   这就是 LeCu所说的 JEPA 模型。它会把最开始的图像等模态的信息，先 encode 到 latent space 里面。这个 latent space 能够更好地模拟人脑的注意力机制，然后我们直接在 latent space 上进行预测。
   我们的最终目标不是把这个 latent space 重新还原成真实世界再去求 loss，而是把我们真实想要生成的东西也 encode 到 latent space 里面，然后只需要在 latent space 里面让 loss 尽量变小。

如果我们能够找到一个很好的 latent space，它就能在里面注意到很多有价值的信息。但这个 latent space 具体怎么得到，目前还非常需要思考，也是很多工作正在探索的方向。

我也在想，如果是 World Action Model 这种方向，我们该怎么样设计一个比较好的 latent space，从而让它能够更好地指导机器人的动作呢？

### Oral 1F Vision-Language Models & Video
这次 oral 里面有两篇文章，我比较感兴趣。
#### Are VLMs Seeing or Just Saying? Uncovering the Illusion of Visual Re-examination
这篇文章是说，他们发现在 VLM 说“让我重新看看这张图”的时候，实际上并没有把太多的注意力放在原图上。

他们做了一个实验：让 VLM 对着一整个数学题做一会儿，然后偷偷把那张图片换掉。结果发现，换掉图片以后 VLM 根本不会发现问题，而是会接着之前的步骤继续往下做。后续发现，VLM 很多时候只是“光说了，但没有做”。这一点在后续的训练中可能需要更加注意，想想怎么去避免这样的问题。

虽然这篇文章实际上对我的工作来说没有什么直接用处，但比较特别的是，它注意到了一个很小的现象，而这个现象实际上又挺有用的。作者把这背后的故事和过程都说得挺清楚，就可以做出一篇很有趣的工作，甚至可以拿到 Oral。

这跟我之前想的“写一篇论文必须得做出一个很厉害的模型”还是很不一样的，真的很有启发。

#### [Motion Attribution for Video Generation](https://arxiv.org/abs/2601.08828)
这篇论文还是与 data attribution（数据归因）有关，这个问题我之前就有关注。之前在叶杰平老师组轮转的时候，他们就讨论过归因在机器学习中的应用，其中一个就是 attribution 的计算。虽然那个计算方法太复杂了，所以一般我们做 attribution 还是直接看梯度——即单个数据所带来的梯度，和你实际整体需要的梯度，它们的开发相关性怎么样。

但是这个东西在视频生成上肯定会有问题。比如说，你算出来的梯度可能相对偏高，但这些梯度带来的都是一些很边缘、很不重要的信息，比如纹路、颜色、背景之类的。而很多时候，我们其实更加关注视频生成模型对“动作”的理解。

所以说，这篇论文比较特别的工作在于：
1. 它专门为“动作”这一块设置了一个专属 loss。
2. 针对这个 loss 产生的这一类梯度进行计算。
3. 从而筛选出对动作最有用的一部分数据来进行训练。

实验结果表明，完全使用这部分最好的数据（比如前 10% 的数据）进行训练，效果甚至比使用整个数据集训练还要好，大大减少了训练的数据量。

我觉得这个思路在机器人方面肯定也会有用。我们到底是要让它学怎么动，还是让它学一些不是那么重要的东西？我们可以想方设法把最关注的东西设计成一个专属 loss。这肯定不简单，但至少提供了一个很好的方向，来做 data attribution 的筛选。

这篇论文，我接下来可以继续更深入地看一下。

#### [RoboMME: Benchmarking and Understanding Memory for Robotic Generalist Policies](https://robomme.github.io/index.html)

这篇工作倒不是做了一个更新的 SoTA 模型，而是对 robot 的 memory 这一方面进行了一个比较系统的讲解。

它把 memory 分成了好几种类型，并设计了不同的 memory 使用方式，然后在不同的任务上进行试验，看哪些的表现比较好。最后发现，不同的任务确实需要不同种类的 memory。

这篇论文我还打算接下来继续深入地读一下。

### Invited talk: Causal Inference with Transformer Models
这个因果推断（causal inference）研究也挺有意思的。它涉及到了一个我相比其他学院的人来说更熟悉的领域，就是关于计量经济学因果推断这一块。

它进行的一个推断，其实是与 LLM 相关的一些语气之类的推断（也可以不只是语气，比如在最后我们进行 RLHF 对齐的时候，需要一些指标，它就针对这些指标来进行因果推断，比如使用更温暖的语调会不会更好，更详细的步骤会不会更好）。

它的主要方法是利用了 LLM 里面的随机性。我们在进行 beam search 的时候，可能会有多种可能的输出结果。由于这些输出都是在同样的上下文中获得的，所以我们可以认为其他的变量都是已经被控制住的一样的，唯一的变量就只有这两个输出之间的变化（比如一个输出更长，一个输出更短；或者一个语气更温暖，一个语气更平淡）。

就此，如果检测出来这两个输出的语气温暖程度不同，就可以通过这个区别，来对“语气是否温暖”和“是否会被用户喜爱”来进行因果推断。然后我们再对这种概率进行大数定律的估算，就可以整体估计出来。大概是这个意思。

具体的方法感觉没有特别巧妙，也有很多地方都挺粗糙的。但确实，之前我完全没有想过在大模型中还可以使用这些计量的统计方法来进行因果推断，还是挺有启发的。说不定如果我以后去其他导师组里，或者去哪里读经济的话，也可以继续关注这一方面：如何在大模型的层面来设计这种统计量进行推断。
